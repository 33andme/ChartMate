## 数据模型

### Post（`models.py` 第 117 行）
```
posts 表
├── id
├── user_id     (外键 → users)
├── content     (TEXT)
├── images      (TEXT，存储图片 URL 列表的 JSON 字符串)
├── like_count  (冗余计数字段，避免每次 COUNT 查询)
└── created_at
```

### Comment（`models.py` 第 141 行）
```
comments 表
├── id
├── post_id   (外键 → posts)
├── user_id   (外键 → users)
├── content   (最长 500 字)
└── created_at
```

### PostLike（`models.py` 第 151 行）
```
post_likes 表
├── id
├── user_id   (外键 → users)
├── post_id   (外键 → posts)
└── created_at
UniqueConstraint("user_id", "post_id")  ← 防止重复点赞
```

## API 接口

### 获取帖子列表（`main.py` 第 1173 行）
`GET /api/posts?page=1&page_size=10`

- 按 `created_at DESC` 分页
- 可匿名访问（`get_optional_user`），但登录后会附带当前用户对每条帖子的点赞状态
- 返回邮箱时做脱敏处理：`abc***@qq.com`

```python
# 批量获取点赞状态（一次查询，不是 N+1）
liked_post_ids = {like.post_id for like in likes}
```

### 发布帖子（`main.py` 第 1222 行）
`POST /api/posts` `{ content, images: [] }`

- 内容不能为空（服务端校验）
- 图片列表存为 JSON 字符串：`json.dumps(req.images)`

### 点赞/取消点赞（`main.py` 第 1252 行）
`POST /api/posts/like` `{ post_id }`

切换逻辑：
```
已有点赞记录 → 删除记录，like_count - 1
无点赞记录   → 新增记录，like_count + 1
```

并发安全：捕获 `IntegrityError`（唯一约束冲突），返回"已点赞"而非报错（`main.py` 第 1285 行）。

### 获取评论（`main.py` 第 1291 行）
`GET /api/posts/{post_id}/comments`

按 `id ASC` 排序（时间正序），评论作者邮箱同样脱敏。

### 发表评论（`main.py` 第 1316 行）
`POST /api/posts/comment` `{ post_id, content }`

## 前端实现要点（`static/community.html`）

### Tab 切换
社区页包含"发现"和"聊天"两个 Tab（`switchTab()`），切换时懒加载对应数据。

### 帖子渲染（`renderPost()`）
- 头像用星座符号代替（`['♈','♉',...][post.id % 12]`），无需上传头像
- 图片网格：1张图 → 单列；多张图 → 3列网格
- 点赞使用乐观更新：先改 UI，再发请求，失败不回滚（用户体验优先）

### 加载更多
`page` 变量自增，追加渲染（`loadPosts(append=true)`），不清空已有内容。

## 图片上传（`main.py` 第 1773 行）
`POST /api/upload`

- 支持格式：JPG / PNG / GIF / WebP
- 大小限制：5MB
- 存储路径：`uploads/<uuid>.<ext>`（本地磁盘）
- 返回 `/uploads/<filename>` 路径，前端拼接完整 URL 使用
