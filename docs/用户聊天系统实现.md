# 用户聊天系统实现

## 涉及文件
- `models.py` — `ChatRoom`、`ChatMessage` 表定义（第 181-211 行）
- `main.py` — 聊天相关 API（第 1570-1766 行）
- `static/messages.html` — 聊天页面前端

---

## 数据模型

### ChatRoom（`models.py` 第 181 行）
```
chat_rooms 表
├── id
├── user1_id  (较小的用户 ID，外键 → users)
├── user2_id  (较大的用户 ID，外键 → users)
├── last_message_at  (最后消息时间，用于列表排序)
└── created_at
UniqueConstraint("user1_id", "user2_id")  ← 同一对用户只有一个聊天室
```

### ChatMessage（`models.py` 第 198 行）
```
chat_messages 表
├── id
├── room_id    (外键 → chat_rooms)
├── sender_id  (外键 → users)
├── message_type  (text / image)
├── content
├── is_read    (是否已读)
└── created_at
```

## API 接口

### 创建/获取聊天室（`main.py` 第 1630 行）
`POST /api/chats` `{ target_user_id: int }`

关键设计：强制 `user1_id = min(A, B)`、`user2_id = max(A, B)`，保证同一对用户无论谁发起，都命中同一行记录，依赖数据库 UniqueConstraint 防止重复。

```python
user1_id = min(current_user.id, request.target_user_id)
user2_id = max(current_user.id, request.target_user_id)
existing_room = session.exec(select(ChatRoom).where(...)).first()
if existing_room:
    return {"room_id": existing_room.id, "exists": True}
# 不存在则新建
```

### 获取聊天室列表（`main.py` 第 1578 行）
`GET /api/chats`

查询用户参与的所有聊天室（`user1_id == me OR user2_id == me`），每个聊天室附带：
- 对方用户信息
- 最新一条消息内容
- 未读消息数（`is_read=False AND sender_id != me`）

### 获取消息历史（`main.py` 第 1671 行）
`GET /api/chats/{room_id}/messages?page=1&limit=50`

- 按 `created_at DESC` 分页查询，返回前端时再 `reverse()` 变成正序
- 同时将对方发送的未读消息批量标记为已读（`is_read = True`）

### 发送消息（`main.py` 第 1728 行）
`POST /api/chats/send` `{ room_id, content, message_type }`

权限验证：调用方必须是聊天室的 `user1_id` 或 `user2_id`，否则返回 403。

发送成功后更新 `ChatRoom.last_message_at`，用于聊天列表按最新消息排序。

## 好友系统与聊天室的关系

好友关系即聊天室的存在，没有单独的好友表：

| 操作 | 实现 |
|------|------|
| 添加好友 | `POST /api/friends` → 创建 ChatRoom（`main.py` 第 1344 行） |
| 删除好友 | `DELETE /api/friends/{id}` → 删除 ChatRoom + 所有 ChatMessage |
| 检查好友 | `GET /api/friends/check/{id}` → 查询 ChatRoom 是否存在 |
| 获取好友列表 | `GET /api/friends` → 查询用户参与的所有 ChatRoom |

---

## 已读状态

- 消息写入时 `is_read = False`
- 接收方调用 `GET /api/chats/{room_id}/messages` 时，服务端自动将对方发送的未读消息批量置为 `is_read = True`（`main.py` 第 1697 行）
- 聊天室列表中的未读数通过实时查询 `is_read=False` 计算，无独立计数字段

## 前端实现要点（`static/messages.html`）

- 通过 URL 参数 `?room=<room_id>` 确定聊天室
- 进入页面时加载历史消息，滚动到底部
- 发送消息后乐观更新 UI，再请求后端
- 消息气泡区分自己（右侧）和对方（左侧）：根据 `sender.is_me` 判断
