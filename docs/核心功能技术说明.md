# 核心功能技术说明

## 一、RAG 知识库在系统中的作用

RAG（检索增强生成）解决了大语言模型"知识有限"的问题。在本系统中的运作流程：

1. **管理员上传**占星学知识文档（PDF/Word/TXT）
2. 文档被切分为 500 字的文本块，通过 SiliconFlow `BAAI/bge-m3` 模型向量化，存入 LanceDB 本地向量数据库
3. 用户发起对话时，系统将用户问题同样向量化，从知识库中**检索最相关的 3 段文本**
4. 将检索结果注入到 AI 的 System Prompt 中，作为回答的知识依据

**作用**：让 AI 占星师能够引用专业的占星学知识（行星含义、星座特质、宫位解读等），而不只依赖模型的训练数据，回答更专业、更准确。

## 二、智能体 ReAct 推理循环

ReAct（Reasoning + Acting）是 LangChain Agent 的核心推理机制，让 AI 能够**自主决策调用哪些工具**来完成复杂任务。

**循环步骤：**

```
用户提问
   ↓
[思考] AI 分析问题，决定是否需要工具，需要哪个工具
   ↓
[行动] 调用工具（如查询运势、检索知识库、查用户数据）
   ↓
[观察] 获取工具返回的结果
   ↓
[再思考] 基于结果继续分析，决定是否继续调用工具或直接回答
   ↓
最终回复用户
```

**本系统中的工具列表：**

| 工具 | 触发场景 |
|------|----------|
| `query_user_data` | 用户询问自己的星盘、运势数据 |
| `search_knowledge` | 需要引用占星学专业知识时 |
| `search_web` | 需要查询实时星象信息时（DuckDuckGo） |
| `calculate_fortune` | 用户询问今日运势评分 |

## 三、星盘数据包含的具体内容

用户创建档案时，系统根据**出生时间 + 出生城市（转经纬度）**，通过 flatlib 库计算完整星盘，以 JSON 格式存储在数据库 `profiles.astral_config` 字段中。

### 1. 基础元信息
| 字段 | 示例值 | 说明 |
|------|--------|------|
| `calculated_at` | `"2024-06-10T09:00:00"` | 星盘计算时间（UTC） |
| `birth_city` | `"北京"` | 出生城市 |
| `coordinates.lat` | `39.9` | 出生地纬度 |
| `coordinates.lon` | `116.4` | 出生地经度 |

### 2. 三大核心星座（顶层字段）
| 字段 | 示例值 | 含义 |
|------|--------|------|
| `sun_sign` / `sun_sign_cn` | `"Scorpio"` / `"天蝎座"` | 太阳星座，代表自我与核心性格 |
| `moon_sign` / `moon_sign_cn` | `"Pisces"` / `"双鱼座"` | 月亮星座，代表情感与潜意识 |
| `asc_sign` / `asc_sign_cn` | `"Leo"` / `"狮子座"` | 上升星座，代表外在形象与第一印象 |

### 3. 十二天体位置（`planets` 字段）

每个天体包含 5 个子字段：

| 子字段 | 示例值 | 说明 |
|--------|--------|------|
| `sign` / `sign_cn` | `"Aries"` / `"白羊座"` | 所在星座 |
| `degree` | `14.52` | 在该星座内的度数（0-30°） |
| `emoji` | `"♈"` | 星座表情符号 |
| `retrograde` | `false` / `true` | 是否逆行 |
| `planet_cn` | `"太阳"` | 天体中文名 |

包含的十二天体：

| 天体 | 占星含义 |
|------|----------|
| 太阳（Sun） | 自我意识、生命力、核心性格 |
| 月亮（Moon） | 情感反应、本能、内心世界 |
| 水星（Mercury） | 思维方式、沟通、学习能力 |
| 金星（Venus） | 爱情观、审美、人际关系 |
| 火星（Mars） | 行动力、欲望、竞争心 |
| 木星（Jupiter） | 幸运、扩张、信仰 |
| 土星（Saturn） | 责任、纪律、人生课题 |
| 天王星（Uranus） | 变革、突破、叛逆 |
| 海王星（Neptune） | 直觉、梦想、灵感 |
| 冥王星（Pluto） | 深层转化、重生、权力 |
| 上升点（Asc） | 出生时东方地平线，决定整个宫位体系 |
| 天顶（MC） | 职业方向、社会形象、人生目标 |

### 4. 十二宫位（`houses` 字段）

以上升点为起点，每 30° 划分一宫，用等宫制计算。每宫包含：

| 子字段 | 示例值 | 说明 |
|--------|--------|------|
| `sign` / `sign_cn` | `"Leo"` / `"狮子座"` | 该宫起始所在星座 |
| `degree` | `125.3` | 该宫起始的黄道经度（0-360°） |

十二宫对应的人生领域：

| 宫位 | 领域 |
|------|------|
| 第1宫 | 自我、外貌、性格 |
| 第2宫 | 财富、物质、价值观 |
| 第3宫 | 沟通、兄弟姐妹、短途旅行 |
| 第4宫 | 家庭、童年、根基 |
| 第5宫 | 爱情、创意、娱乐 |
| 第6宫 | 健康、工作、日常习惯 |
| 第7宫 | 婚姻、伴侣、合作 |
| 第8宫 | 转化、遗产、深层心理 |
| 第9宫 | 哲学、远途旅行、高等教育 |
| 第10宫 | 事业、社会地位、声誉 |
| 第11宫 | 友谊、团体、理想 |
| 第12宫 | 潜意识、隐秘、灵性 |

### 5. 每日运势中的行星相位（实时计算，不存储）

运势生成时，系统额外调用 flatlib 获取**当日行星实时位置**，与用户星盘行星对比，检测以下相位并计算加成分值：

| 相位 | 角度 | 容许度 | 影响 |
|------|------|--------|------|
| 合相 | 0° | ±8° | 能量集中，正负由行星性质决定 |
| 六分相 | 60° | ±4° | 和谐，小幸运 |
| 四分相 | 90° | ±6° | 张力，需克服挑战 |
| 三分相 | 120° | ±6° | 和谐，流畅顺利 |
| 对分相 | 180° | ±8° | 对立，需要平衡 |
