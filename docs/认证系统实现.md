# 认证系统实现

## 涉及文件
- `auth.py` — 所有认证逻辑
- `models.py` — `User`、`VerifyCode` 表（第 16-26 行、第 167-174 行）
- `email_service.py` — 邮件发送
- `main.py` — Auth API 路由（第 529-600 行）

---

## 整体流程

```
注册流程：
  发送验证码 → 邮箱收到 6 位数字码 → 提交邮箱+验证码+密码 → 创建用户+签发 Token

登录流程：
  提交邮箱+密码 → 验证密码哈希 → 签发 Token
```

## 1. 验证码（`auth.py` 第 113-151 行）

### 生成（`generate_verify_code()`）
- 生产环境：`random.choices(string.digits, k=6)` 生成 6 位随机数字
- 测试环境（`ENV=test`）：固定返回 `123456`，方便开发调试

### 发送（`send_verify_code()`）
1. 清除该邮箱所有旧验证码（防止多条有效码并存）
2. 生成新验证码，有效期 10 分钟，写入 `verify_codes` 表
3. 调用 `email_service.send_verification_code_email()` 发送邮件
4. 返回 code（测试环境下接口响应中也会返回 `debug_code`）

### 验证（`validate_verify_code()`，第 154 行）
按顺序检查：
1. 数据库中是否存在该邮箱的验证码记录
2. 是否已被使用（`is_used = True`）
3. 是否已过期（`datetime.utcnow() > expires_at`）
4. 验证码字符串是否匹配

验证通过后立即将 `is_used` 置为 `True`，防止重复使用。

## 2. 密码存储（`auth.py` 第 37-42 行）

使用 `passlib` + `bcrypt` 单向哈希，不存储明文：

```python
pwd_context = CryptContext(schemes=["bcrypt"], deprecated="auto")
hash_password(password)    # → bcrypt 哈希字符串
verify_password(plain, hashed)  # → bool
```

> 注意：`bcrypt 4.x` 删除了 `__about__` 属性，`auth.py` 第 20 行有兼容补丁：
> ```python
> if not hasattr(_bcrypt, '__about__'):
>     _bcrypt.__about__ = type('_about', (), {'__version__': _bcrypt.__version__})()
> ```

## 3. JWT Token（`auth.py` 第 49-69 行）

### 签发（`create_access_token()`）
```python
payload = {
    "sub": str(user_id),   # 用户 ID
    "email": email,
    "exp": utcnow + 7天,
}
jwt.encode(payload, SECRET_KEY, algorithm="HS256")
```

有效期 7 天（`ACCESS_TOKEN_EXPIRE_HOURS = 24 * 7`）。

### 验证（`decode_token()`，第 59 行）
`jwt.decode()` 自动校验签名和过期时间，失败抛出 `HTTP 401`。

## 4. FastAPI 依赖注入（`auth.py` 第 76-106 行）

### `get_current_user()`
- 从请求头 `Authorization: Bearer <token>` 提取 Token
- 解码后从数据库查询用户对象
- Token 缺失或无效均返回 `HTTP 401`
- 用法：`current_user: User = Depends(get_current_user)`

### `get_optional_user()`
- 与上相同，但 Token 缺失或无效时返回 `None` 而非报错
- 用于社区帖子列表等"可匿名访问但登录后有额外功能"的接口

## 5. 注册/登录接口（`main.py` 第 529-573 行）

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/auth/send-code` | POST | 发送验证码，测试环境返回 `debug_code` |
| `/api/auth/register` | POST | 验证码 + 密码注册新用户 |
| `/api/auth/login` | POST | 邮箱 + 密码登录 |
| `/api/auth/me` | GET | 返回当前登录用户信息 |
| `/api/auth/check-email` | GET | 检查邮箱是否已注册 |

所有成功响应均返回 `{ token, user: { id, email } }`，前端存入 `localStorage`。

---

## 6. 邮件发送（`email_service.py`）

- 支持 SSL（端口 465）和 TLS（`starttls()`）两种连接方式，由 `SMTP_USE_SSL` 环境变量控制
- 同时发送纯文本和 HTML 两种格式（`MIMEMultipart('alternative')`），邮件客户端自动选择
- 测试环境（`ENV=test`）跳过实际发送，仅打印日志

## 环境变量配置

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `SECRET_KEY` | `astro-secret-key-...` | JWT 签名密钥，**生产环境必须替换** |
| `ENV` | `test` | 设为 `production` 关闭固定验证码和 debug 接口 |
| `SMTP_SERVER` | `smtp.qq.com` | 邮件服务器 |
| `SMTP_USER` | — | 发件邮箱 |
| `SMTP_PASSWORD` | — | 邮箱授权码 |
